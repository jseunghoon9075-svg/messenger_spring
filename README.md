**LLM 통합 개요**

  프로젝트는 일반 채팅 API를 넘어
  서버 측에서 LLM(대규모 언어 모델)을 통합하여
  맥락 기반 AI 응답 생성 기능을 제공합니다.
  
  LLM은 stateless 특성 때문에
  서버가 직접 대화 컨텍스트를 재구성 및 관리하며 호출하는 구조입니다.

**문제 정의**

  LLM을 단순히 호출하면 다음과 같은 문제가 발생합니다:

    최근 대화만 전달 시 → 맥락 손실
    전체 대화 전달 시 → 토큰 초과 / 높은 비용
    대화 길이가 길어질수록 → 응답 지연

  이를 해결하기 위해 서버에서 요약 + 최근 대화 기반 토큰 관리 전략을 설계했습니다.

**해결 전략**

  서비스는 다음과 같은 흐름으로 동작합니다:
  
  1. 사용자가 메시지를 전송하면 DB에 저장
  2. 전체 메시지가 일정 개수(예: 20개)를 넘으면 자동 요약 생성
  3. 요약 내용 + 최근 메시지를 조합하여 LLM에 입력
  4. 모델의 응답을 받아 DB에 저장
  5. WebSocket으로 실시간 응답 전송

OpenAI API 응답에 포함된 usage.total_tokens를 파싱하여:

  - 토큰 기준을 넘을 때 자동 요약 트리거
  - 기존 요약 + 신규 메시지 결합 재요약
  - 불필요한 토큰 소모를 방지
  
이렇게 함으로써 비용 최적화 및 안정적인 컨텍스트 전달을 구현했습니다.

**아키텍처 개요**

User Client (React/Vue)
        ↓ WebSocket or REST
Spring WebSocket/REST Controller
        ↓ ChatService (비즈니스 로직 담당)
(1) 메시지 저장 → DB
(2) Summary 판단 → 요약 저장
(3) Context 조합 → LLM 호출
        ↓ AI 응답 수신
        ↓ DB 저장
        ↓ WebSocket으로 실시간 전송

Service 계층은:

  - 트랜잭션 경계 관리
  - 예외 처리
  - 외부 API 호출 책임

DB 계층은 순수 저장 및 조회에 집중하도록 분리되어 있습니다.

**설계 의도 및 고려 요소**

  - Stateless LLM을 고려해 서버가 상태를 관리
  - 토큰 한계와 비용 문제를 동시에 해결
  - 확장성을 고려한 계층 분리 (Service / Repository / Controller)

즉, 단순 코드 연동이 아니라 아키텍처 설계 기반 LLM 통합입니다.
